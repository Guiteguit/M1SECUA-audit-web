# TP5 — Secure Coding

## Objectif

Rédiger une fiche de recommandations développeur.

## Consigne

Choisir 3 vulnérabilités et rédiger une recommandation de secure coding pour chacune.

Exemples :

- SQL Injection → requêtes préparées ;
- XSS → encodage de sortie / textContent ;
- JWT → validation stricte de signature ;
- IDOR → vérification d’autorisation côté serveur ;
- Secrets → coffre de secrets ;
- Désérialisation → éviter les formats dangereux.

## Format attendu

Pour chaque recommandation :

- mauvaise pratique ;
- risque ;
- bonne pratique ;
- exemple de pseudo-code corrigé ;
- test de vérification.
