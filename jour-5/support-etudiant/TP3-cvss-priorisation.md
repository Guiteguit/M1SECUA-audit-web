# TP3 — CVSS et priorisation

## Objectif

Classer les vulnérabilités selon leur sévérité et leur priorité de traitement.

## Consigne

Pour chaque finding critique ou élevé, compléter :

- Attack Vector ;
- Attack Complexity ;
- Privileges Required ;
- User Interaction ;
- Confidentiality ;
- Integrity ;
- Availability ;
- justification.

## Important

La sévérité CVSS n’est pas toujours égale à la priorité projet.

Exemple :

- une faille critique sur une fonctionnalité non exposée peut être moins urgente ;
- une faille high exposée publiquement peut être prioritaire ;
- la présence de données personnelles change la priorité.
